一.谈谈什么是数据库，DBMS，关系型数据库

1.数据库

字面意思，按照数据结构来组织、存储和管理数据的仓库，是系统的数据集合。

2.DBMS

即数据库管理系统，是一种能让用户对数据库中存放的数据进行建立、修改、删除、查找等操作的软件程序，能将计算机中具体的物理数据转化为抽象逻辑数据，降低数据库管理门槛，mysql即是一款数据库管理系统。

它有几种主要类型

Hierarchical Database, HDB：最基本的数据关系是层次关系，采取存储数据的父子关系，大概比较像pstree吧，它配置设置存储为具有节点的树结构。

network dbms : 数据和资源共享这两种技术结合在一起，以数据为基础，加上一些前台计算机程序，eg.RDM 服务器（远程服务器管理）

Relational Database, RDB：详见下面

Object Oriented Database，OODB：要求存储的数据是对象的类型，以对象为单位进行管理，面对对象数据库就是用来保存这些对象的数据库。

3.关系型数据库

以表格形式定义数据库关系，结构化强类型，使用SQL（Structured Query Language，结构化查询语言）对数据进行操作，比如mysql。它格式一致都是表结构，可用于复杂查询，但在海量数据面前，读写性能可能会有一点差。

三.谈谈主键与外键

1.主键

（1）主键

Primary Key，某一列的值能够唯一标识着表中每一行数据，每一张表只能有一个主键，不允许有空值，主键索引是一种特殊的唯一索引

（2）"二.4"再次插入数据1失败原因

因为我选择了id作为主键，新增数据为重复数据，它的id=1，在之前插入的数据中已经存在id=1的数据，如果它能成功插入，那主键就不能再唯一标识，因为主键强调唯一性，所以不可能在主键列中出现两个相同值。

（3）为什么设置它为主键

因为我认为在company表中，id的特征性是最强的，名字可能重名，时间可能相同（虽然这张表中没有），但是它们的id一定是不同的，一定是唯一的，更符合主键的唯一标识性，后面employee表的主键也选择了id，是同样的原因

2.外键

（1）外键

外键就是作用于两个表数据之间的链接的一列或多列，用来保证表与表之间的数据的完整性和准确性。

一般情况下，是一个表的外键关联另一个表的主键来构成联系，所选两个字段的字段类型一定要相同。

（2）“二.5”中插入数据失败的原因

想插入的数据所在公司id为4（即我表中的id_of_company=4)，该公司id在外键连接的company表中不存在，所以不能插入。

举个例子（其实就是它的现实意义），插入外键的那张表是一个个员工1，2，3，外键连接的另一张表是各个公司1，2，3。现实生活中，不可能凭空出现一个员工加入了公司4，因为根本没有这个公司，自然无法加入，所以该数据插入一定会失败。

但是，如果在company表中插入id=4的新公司id数据，并不会失败，因为现实生活中一个公司一开始的建立，并不需要一开始就有员工存在，也就是说，员工是否存在无法约束公司能否新建，但公司是否存在一定约束员工能否加入。

（3）为什么设置它为外键

我设置了id_of_company为外键，与company表中主键公司id意义相同且对应，只有它能建立起两张表的对应连接关系，两种表唯一相同的一列就是公司id，举个例子，玩拼图的话，两张表各为拼了一部分的小拼图，只有这两块对应的拼图（两个公司id）可以完全拼在一起，连接两个拼了一部分的小拼图，使拼图更加完整有关联。它可以保证我在employee表中找的每一行数据，都能通过这个公司id，精准对标到另一张表中，获得更多的对应正确的数据。外键其实就相当于一座桥，和后续查询语句的inner join构成的联系感觉很像。
